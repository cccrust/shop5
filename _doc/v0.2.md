# Shop5 v0.2 — Web API

在 v0.1 CLI 基礎上加入 RESTful Web API，使用 axum 框架提供 HTTP 介面。

## 新增技術棧

- **Web 框架:** axum 0.8
- **非同步執行:** tokio
- **HTTP 伺服器:** Tower (axum 內建)
- **CORS:** tower-http

## 資料庫 (維持 5 張表)

與 v0.1 相同，無結構變動。Web API 直接操作同一 SQLite 資料庫。

## RESTful Web API

所有端點前綴 `/api`，回傳 JSON。

### 使用者

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/users` | 列出使用者 (支援 `?search=`) |
| POST | `/api/users` | 建立使用者 |
| GET | `/api/users/{id}` | 取得單一使用者 |
| PUT | `/api/users/{id}` | 更新使用者 |
| DELETE | `/api/users/{id}` | 刪除使用者 |

### 商品

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/products` | 列出商品 (支援 `?seller_id=&status=`) |
| POST | `/api/products` | 建立商品 |
| GET | `/api/products/{id}` | 取得單一商品 |
| PUT | `/api/products/{id}` | 更新商品 |
| DELETE | `/api/products/{id}` | 刪除商品 |

### 購物車

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/cart/{user_id}` | 列出購物車 |
| POST | `/api/cart` | 加入購物車 |
| DELETE | `/api/cart` | 移除購物車項目 |
| DELETE | `/api/cart/{user_id}` | 清空購物車 |

### 訂單

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/orders` | 列出訂單 (支援 `?buyer_id=&seller_id=`) |
| POST | `/api/orders` | 從購物車建立訂單 |
| GET | `/api/orders/{id}` | 取得訂單 (含明細) |
| PUT | `/api/orders/{id}` | 更新訂單狀態 |

### 錯誤處理

| HTTP 狀態碼 | 情境 |
|------------|------|
| 404 | 資源不存在 |
| 400 | 參數錯誤（價格為負、庫存不足等） |
| 500 | 伺服器內部錯誤 |

錯誤回傳格式：`{"error": "..."}`

### AppState

使用 `Arc<Mutex<Connection>>` 跨執行緒共享 SQLite 連線。

## 執行方式

```bash
cargo run -- web --port 8080         # 啟動 API (預設 0.0.0.0:8080)
./run.sh api                         # 透過 run.sh 啟動
```
