# v1.0 認證系統 — 正式版

## 目標
完整的帳號密碼認證系統，支援註冊、登入、登出、Token 驗證，並將使用者身份注入所有需要授權的 API 端點。前端全面改用 AuthContext，取代原本的 CurrentUserContext。

## 技術選型
- **密碼雜湊:** `bcrypt` v0.16（`DEFAULT_COST = 12`）
- **Token 生成:** `uuid` v1（v4 隨機 UUID）
- **Token 儲存:** `sessions` 資料表（server-side，支援撤銷與過期）
- **Token 傳遞:** `Authorization: Bearer <token>` header
- **Token 過期:** 7 天（`datetime('now', '+7 days')`）

## Web API 新增

### 認證端點

| 方法 | 路徑 | 說明 | 驗證 |
|------|------|------|------|
| POST | `/api/auth/register` | 註冊新使用者 | 無 |
| POST | `/api/auth/login` | 登入（回傳 token + user） | 無 |
| POST | `/api/auth/logout` | 登出（清除該使用者所有 session） | Bearer |
| GET | `/api/auth/me` | 取得當前使用者資訊 | Bearer |

**註冊請求:**
```json
{ "username": "charlie", "display_name": "查理", "email": "c@test.com", "password": "mypass", "role": "buyer" }
```

**登入請求:**
```json
{ "username": "alice", "password": "pass123" }
```

**登入/註冊回應:**
```json
{
  "token": "bd64d50b-3c11-4fac-8931-24bbfbedaa14",
  "user": { "id": 1, "username": "alice", "display_name": "愛麗絲", "email": "alice@test.com", "role": "seller", ... }
}
```

### /me 端點（需 Bearer Token）

所有需要使用者身份的端點新增 `/me` 版本，自動從 token 推斷 user_id：

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/cart/me` | 我的購物車 |
| DELETE | `/api/cart/me` | 清空我的購物車 |
| PUT | `/api/cart/me/{product_id}` | 更新購物車數量 |
| GET | `/api/orders/me` | 我的訂單列表 |
| POST | `/api/orders/me` | 從購物車建立訂單 |
| POST | `/api/orders/me/preview` | 預覽訂單 |
| GET | `/api/seller/me/orders` | 賣家訂單管理（限 seller） |
| GET | `/api/seller/me/products` | 賣家商品管理（限 seller） |
| GET | `/api/seller/me/stats` | 賣家儀表板統計（限 seller） |

非 seller 存取 `/seller/me/*` 回傳 **403 Forbidden**。

### 錯誤碼

| 狀態碼 | 情境 |
|--------|------|
| 401 | 缺少/無效 Token、帳號或密碼錯誤 |
| 403 | 非賣家存取賣家端點 |

### case9.sh 測試涵蓋

1. 建立含 email/password 的使用者
2. 註冊新使用者（回傳 token）
3. 正確登入、錯誤密碼 401、不存在使用者 401
4. GET /auth/me（有效 token → 200；無 token → 401）
5. 購物車 /me 端點
6. 訂單 /me 建立 + 列表
7. 賣家 /me 端點（訂單、商品、統計）
8. 非賣家存取賣家端點 → 403
9. 登出後 token 失效 → 401
